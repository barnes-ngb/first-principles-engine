# Seed Vault v1 — First Kit Build Runbook

**Goal:** Hand-build ONE polished, non-custom Seed Vault starter kit using FPE tools as they ship
plus simple by-hand pieces, then test it with 3–5 warm families. No app work — pure Track A proof.

**The one prerequisite:** re-enroll HP Instant Ink (300-page plan). You'll print a lot during
iteration, and the unit economics depend on it.

**Roles** (all of it logs as school via the creative timer — Art / Language Arts / Practical Arts):
- **London — creative director:** invents the world, draws, picks AI scenes, approves art.
- **Lincoln — operations:** QA against a visual checklist, count, cut, pack.
- **Nathan — producer:** printing, final quality control, selling, safety.

**Kit contents:** 6-page story · 1 sticker sheet · 5 clue cards · 1 defense map · 1 badge ·
1 parent setup page.

**Materials to have on hand:** white printable sticker paper, cardstock (for cards + badge),
envelopes or poly sleeves, labels. All cheap; one trip covers the first several kits.

---

## Phase 0 — Set the world (London)

Before any tool, London invents the kit's world in a creative-director session — capture by
drawing and/or voice:
- Name the silly garden zombies, the plant defenders, and the treasure (the "Seed Vault").
- **Keep it original** — invent names like *Sprout Cannon*, *Blocky Bog Monster*, *Pea Blaster*.
  Avoid Plants vs. Zombies and Minecraft names/characters. (The app's copyright rewrite helps, but
  the invented content should be his own.)

This becomes the source material for every piece below.

---

## Phase 1 — The story (My Books generator) → *verifies Story Guide + Print*

1. Open **My Books → Story Guide** (the voice-guided 5-question wizard — using it here also
   verifies the Story Guide handoff for TEST-03).
2. **Theme:** pick a clean, non-branded theme (Adventure / Fantasy / Animals). **Not Minecraft** —
   the dedicated "Block Garden" theme is July work (FEAT-28); a clean existing theme is fine for v1.
3. Feed London's world: hero = a kid garden defender, problem = silly garden zombies invading,
   resolution = plant defenders + reaching the Seed Vault. ~6 pages.
4. Generate → illustrations land per page. **London reviews and approves the scenes.**
5. Export → **mini-book PDF** (half-letter 5.5×8.5). This exercises the print path (TEST-03 — watch
   for the new skipped-image toast).

---

## Phase 2 — The sticker sheet (Sticker pipeline) → *verifies sticker generation*

1. Generate stickers of the plant defenders, silly zombies, and treasure icons (transparent).
   Aim for ~12–20 on one sheet.
2. Tag and save to the **Sticker Library** (verifies sticker generation for TEST-03).
3. Print the sheet on **white sticker paper**.
4. **Cut** (Lincoln/London — hand-cut for v1; a Cricut speeds kiss-cutting later if volume grows).
   This is London's "place/sort stickers" job.

---

## Phase 3 — Clue cards · defense map · badge (by hand for v1)

These aren't tool-produced yet (that's the July kit assembler, FEAT-27), so v1 is made by hand in
Canva or from London's own drawings — cheap, and squarely the creative-director job.

- **5 clue cards** — one instruction per card, big visuals, minimal text: find-the-symbol,
  color-match, simple counting, arrow paths. (Accessibility-first: short, icon-driven, no long
  paragraphs, nothing requiring the child to explain themselves.)
- **1 defense map** — a simple path or grid where stickers get placed along the route to the Seed
  Vault. London draws it or uses a Canva template.
- **1 badge / certificate** — "Seed Vault Defender," on cardstock. London's art or Canva.

---

## Phase 4 — Parent setup page

One page, plain: a 30-second "what this is," how to set up (hide the clue cards, lay out the map),
age 5+ / small-parts note, and the ~20–45 minute play estimate. *(I can draft this copy once
London's world is named — just send me the names.)*

---

## Phase 5 — Assembly + QA (Lincoln operations)

Print a visual production checklist and run each kit through it:

- [ ] Story booklet printed
- [ ] Sticker sheet printed + cut
- [ ] 5 clue cards printed + cut
- [ ] Defense map printed
- [ ] Badge printed
- [ ] Parent setup page printed
- [ ] All pieces counted
- [ ] Packed in envelope / sleeve
- [ ] Label added
- [ ] Quality-checked

---

## Phase 6 — Test with 3–5 families

Frame it as *"we're testing a kids adventure kit and need a few families' feedback"* — not
"support my son's business." Ask each:

1. Did your child understand what to do?
2. How long did it hold their attention?
3. Was setup easy? What confused them?
4. What part did they like best?
5. Would you pay $15 printed / $8 PDF?
6. Would this work at a birthday party?

Watch for: did kids actually use the stickers; did they need too much parent help; did they care
about the story; did the map and clue cards land.

---

## Success gate

A kit passes v1 when: a parent gets it in ~30 seconds, a child can start without a long
explanation, play lasts 20–45 minutes, and it photographs well. Then improve, photograph properly,
and move toward selling 10–20 locally.

---

## What this build also accomplishes

Making the kit exercises the exact features TEST-03 needs verified on a real tablet — Books
generator, Story Guide, sticker generation, and print. So this *is* your pipeline verification.
Note any bugs back to the design chat (print toast, sticker-gen failures, theme leakage) and
they become tracked closure items.

It also feeds July: whatever's clunky to do by hand here (clue cards / map / badge) is precisely
the spec for the FEAT-27 kit assembler.
